  <name>Task 1: Create backend Deployment and Service manifests</name>
  <files>k8s/base/backend/deployment.yaml, k8s/base/backend/service.yaml</files>
  <action>
Create directory `k8s/base/backend/`.

Create `k8s/base/backend/deployment.yaml`:
- apiVersion: apps/v1, kind: Deployment
- metadata: name `backend`, namespace `multica`, labels `app.kubernetes.io/name: backend` and `app.kubernetes.io/part-of: multica`
- spec.replicas: 2
- spec.selector.matchLabels: `app.kubernetes.io/name: backend`
- spec.strategy: type RollingUpdate, maxSurge 1, maxUnavailable 0
- Pod template labels: same as selector plus `app.kubernetes.io/part-of: multica`
- spec.template.spec.serviceAccountName: backend-sa
- spec.template.spec.terminationGracePeriodSeconds: 30
- Single container named `backend`:
  - image: `backend:latest` (placeholder — overlays replace with real registry)
  - ports: containerPort 8080, name `http`
  - command: ["./server"] (override entrypoint.sh which also runs migrations)
  - envFrom:
    - configMapRef: name `backend-config`
    - secretRef: name `backend-secrets`
    - secretRef: name `postgres-secrets`
  - resources:
    - requests: cpu 100m, memory 128Mi
    - limits: cpu 500m, memory 512Mi
  - livenessProbe:
    - tcpSocket: port 8080
    - periodSeconds: 10
    - failureThreshold: 3
  - readinessProbe:
    - httpGet: path /health, port 8080
    - periodSeconds: 5
    - failureThreshold: 3
  - startupProbe:
    - httpGet: path /health, port 8080
    - periodSeconds: 2
    - failureThreshold: 30
  - lifecycle:
    - preStop:
      - exec: command ["sleep", "10"]

Create `k8s/base/backend/service.yaml`:
- apiVersion: v1, kind: Service
- metadata: name `backend`, namespace `multica`, labels `app.kubernetes.io/name: backend` and `app.kubernetes.io/part-of: multica`
- spec.type: ClusterIP
- spec.selector: `app.kubernetes.io/name: backend`
- spec.ports: port 8080, targetPort 8080, protocol TCP, name `http`

Follow the EXACT same YAML formatting and structure as the Phase 1 postgres manifests (StatefulSet and Service).

  <verify>
    <automated>kustomize build k8s/base/backend/ 2>&1 | head -5</automated>
  </verify>

    - Both files are valid YAML that kustomize can process
  </done>
